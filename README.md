# D-ask AI & Backend

**학교 문서 통합 요약 및 지능형 Q&A 시스템**

D-ask는 학교 홈페이지 등 여러 곳에 파편화되어 있는 학교 관련 문서들을 한곳에 모아 AI가 자동으로 요약하고, 사용자의 질문에 즉각 답변해 주는 서비스의 시스템입니다.

---

## 프로젝트 동기 (Motivation)

* **정보 파편화**: 학교 관련 정보가 사이트, 학교 홈페이지 등으로 나누어져 있어 확인이 번거로움.
* **검색의 어려움**: 필요한 정보를 찾기 위해 방대한 양의 문서를 하나하나 대조해야 하는 불편함 발생.
* **접근성 개선**: "이 정보를 한곳에서 편하게 볼 수는 없을까?"라는 고민에서 시작되었습니다.

## 프로젝트 목적 (Objectives)

1. **자동 요약**: 긴 학사 규정이나 공지사항을 핵심 내용 위주로 요약하여 정보 파악 시간을 단축합니다.
2. **지능형 QnA**: 문서 내용에 기반한 질문 답변 기능을 통해 사용자의 궁금증을 즉시 해결합니다.
3. **효율성 증대**: 반복적인 질문에 대한 대응 시간을 줄이고, 정보 접근성을 획기적으로 개선합니다.

---

## 🛠 주요 기술 및 기능 (AI & Backend)

### 1. RAG (Retrieval-Augmented Generation) 기반 Q&A
* 사용자의 질문이 들어오면 `ChromaDB`에서 관련 문서 조각을 검색합니다.
* 검색된 맥락(Context)을 바탕으로 LLM이 정확한 답변을 생성합니다.



### 2. 자동 문서 요약 (Summarization)
* 수집된 학교 문서의 핵심 내용을 추출하여 대시보드에 제공합니다.
* 긴 텍스트 데이터에서 불필요한 정보는 거르고 핵심 키워드 중심의 정리를 수행합니다.

### 3. 멀티 소스 데이터 크롤러 (Crawler)
* 학교 공식 홈페이지 및 외부 커뮤니티의 데이터를 정기적으로 수집합니다.
* 수집된 데이터는 전처리 과정을 거쳐 벡터 데이터베이스에 저장됩니다.

---

## 주요 디렉토리 구조

* `ai/`: 프롬프트 엔지니어링, LLM 체인 로직 (LangChain 등)
* `backend/`: API 엔드포인트 및 서버 비즈니스 로직
* `crawler/`: 웹 사이트 및 SNS 정보 수집 스크립트
* `chroma.db/`: 문서 임베딩 데이터가 저장되는 벡터 데이터베이스
* `data/`: 원본 문서 파일(PDF, json 등) 보관함

---

## 기술 스택 (Tech Stack)

* **Language:** Python
* **Framework:** FastAPI
* **AI Engine:** LangChain
* **Vector DB:** ChromaDB
* **Data Scraping:** BeautifulSoup
